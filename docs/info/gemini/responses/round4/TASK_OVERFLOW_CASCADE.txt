4. Evaluation of Propagation Strategies
When a node at coordinate $\mathbf{x}$ experiences an overflow (energy $E > E_{max}$), the excess energy $\Delta E$ must be moved. We evaluated three distinct strategies for this transport mechanism.
4.1 Strategy A: Spatial Propagation (Neighbor Spillover)
This strategy treats the Nikola grid like a fluid surface. If a bucket (node) overflows, the fluid spills into the adjacent spatial buckets (neighbors).
* Mechanism: Excess $\Delta E$ at node $(x,y,z)$ is distributed to neighbors $(x \pm 1, y, z)$.
* Physical Analogy: Diffusion or heat conduction.
* Analysis:
   * Locality: High. Preserves the general spatial location of the energy.
   * Semantic Integrity: Low. In the Nikola Model, spatial position encodes semantic meaning (e.g., "Apple" is at $x=10$, "Pear" is at $x=11$).1 Spilling energy from "Apple" to "Pear" effectively blurs the concepts. It reduces the semantic resolution of the system, turning sharp concepts into fuzzy blobs.
   * Stability Risk: High. If a cluster of nodes represents a strong concept, they are likely all near saturation. Spilling energy to a neighbor that is also full triggers an immediate secondary overflow. This creates a spatial shockwave—a "Grey Goo" scenario where the overflow expands outward at the speed of light ($c$), consuming the grid.
4.2 Strategy B: Global Dissipation (Entropy Sink)
This strategy treats the excess energy as waste heat to be immediately removed from the system.
* Mechanism: $\Delta E$ is subtracted from the node and added to a global EntropyLog. The node is clamped to $\pm 4$.
* Physical Analogy: Radiative cooling.
* Analysis:
   * Stability: Absolute. Energy is removed, making the system strictly dissipative. It guarantees no Epileptic Resonance.
   * Information Preservation: Zero. The magnitude of the overflow is lost. The system "forgets" that a strong interaction occurred. In a cognitive context, this prevents the formation of "Eureka moments"—high-energy convergence events that signal a breakthrough. The system becomes overly stable and uncreative.
4.3 Strategy C: Dimensional Propagation (Spectral Cascading)
This strategy utilizes the high-dimensional topology of the $T^9$ manifold. If the Spatial X dimension is full, the energy pushes into the Spatial Y dimension, or the Quantum U dimension.
* Mechanism: Excess energy at dimension $D_i$ of node $\mathbf{n}$ promotes to dimension $D_{i+1}$ of the same node $\mathbf{n}$.
* Physical Analogy: Electron excitation. An electron in a full valence shell jumps to a higher energy orbital.

   * Capacity: The 9-dimensional phase space offers vast room for local energy storage before spatial spreading is necessary.

To balance stability with information density, the Nikola Model adopts a hybrid of Strategy B and Strategy C, termed Two-Phase Spectral Cascading with Saturation.1
1. Phase 1 (Conservation): Attempt Dimensional Propagation. If Dimension $i$ overflows, generate a Carry Soliton to Dimension $i+1$. This prioritizes information retention.

It adheres to the principle of Thermodynamic Constitutionalism 1: the laws of physics (stability) override the desire for computation (overflow).

5.1 System Definitions
Let the state of a node be a vector $\mathbf{V} \in \mathbb{Z}^9$, where each component $V_i$ represents the value of dimension $i$.

Let $\Delta_{in}$ be an input perturbation (e.g., adding +1).
The update rule for a dimension $i$ receiving an input $C_{in}$ (carry in) is:




$$V_i' = V_i + C_{in}$$
If $V_i' \notin S$, an overflow occurs.
Let $C_{out}$ be the carry generated by this overflow.
Let $R_i$ be the remainder left in dimension $i$.
5.2 The Saturating Carry Operator
The Nikola specification defines a specialized operator for the propagation, denoted as $\oplus_{sat}$.
Let $C_i$ be the carry generated by dimension $i$ destined for $i+1$.
Standard Carry Generation (Phase 1):




$$C_i = \lfloor (V_i + C_{in}) / 9 \rfloor_{centered}$$


$$R_i = (V_i + C_{in}) \pmod 9_{centered}$$
Saturating Application (Phase 2):
The application of $C_i$ to the next dimension $V_{i+1}$ is conditional:


$$V_{i+1}^{new} = \begin{cases} V_{i+1} + C_i & \text{if } |V_{i+1} + C_i| \le 4 \\ V_{i+1} & \text{if } |V_{i+1} + C_i| > 4 \quad (\text{Dissipation Condition}) \end{cases}$$
If the Dissipation Condition is met:
1. The energy $E_{diss} \leftarrow E_{diss} + |C_i|$ is logged.
2. The next carry $C_{i+1}$ is forced to $0$.
5.3 Proof of Boundedness
Theorem: For any initial state $\mathbf{V}$ and any finite input perturbation, the sequence of carries $C_0, C_1, C_2, \dots$ generated by the Two-Phase logic terminates after at most $N=1$ step relative to the source of the carry.
Proof:
Consider a cascade initiating at dimension $k$.
An input causes $V_k$ to overflow, generating carry $C_k \neq 0$.
The logic attempts to apply $C_k$ to dimension $k+1$.
There are only two possible states for dimension $k+1$:
Case 1: The Target is Receptive (Unsaturated)




$$|V_{k+1} + C_k| \le 4$$


In this case, the carry is successfully absorbed into $V_{k+1}$.
The new value $V_{k+1}^{new}$ is within the valid range $[-4, 4]$.
Since $V_{k+1}^{new}$ is valid, no overflow occurs at $k+1$.
Therefore, the next carry $C_{k+1} = 0$.
The chain terminates. Length = 1.
Case 2: The Target is Refractory (Saturated)




$$|V_{k+1} + C_k| > 4$$


This occurs, for example, if $V_{k+1} = 4$ and $C_k = 1$.
The Dissipation Condition triggers.
The value $V_{k+1}$ is left unchanged (clamped at its max).
The energy of the carry $C_k$ is transferred to the entropy sink.
No addition takes place at $k+1$, so no new overflow is generated.
Therefore, the next carry $C_{k+1} = 0$.
The chain terminates. Length = 1.
Conclusion:
In both physically possible scenarios, the generation of a carry $C_{k+1}$ is impossible. A carry generated at $k$ is either absorbed or dissipated at $k+1$. It never propagates to $k+2$.
Thus, the "Carry Avalanche" (infinite recursion) is mathematically precluded by the saturation logic. The maximum computational complexity for handling an overflow is strictly $O(D)$, where $D=9$ is the dimensionality of the manifold.

namespace nikola::core {

   constexpr int8_t MAX_NIT = 4;
   constexpr int8_t MIN_NIT = -4;
   constexpr int NUM_DIMS = 9;

   struct NonaryDigit {
       int8_t value; // Range [-4, 4]
       
       bool is_saturated_positive() const { return value >= MAX_NIT; }
       bool is_saturated_negative() const { return value <= MIN_NIT; }
   };

   /**
    * @brief Performs addition with Two-Phase Spectral Cascading.
    *        Manages overflow via dimensional propagation or dissipation.
    * 
    * @param target The node state vector (in/out).
    * @param input The perturbation vector to add.
    * @param entropy_sink Logger for thermodynamic accounting.
    */
   void handle_overflow(
       std::array<NonaryDigit, NUM_DIMS>& target, 
       const std::array<NonaryDigit, NUM_DIMS>& input,
       EntropyLog& entropy_sink
   ) {
       std::array<int8_t, NUM_DIMS> pending_carries = {0};

       // --- PHASE 1: Local Addition & Carry Generation ---
       for (int i = 0; i < NUM_DIMS; ++i) {
           // 1. Calculate Ideal Sum
           int sum = target[i].value + input[i].value; 

           // 2. Check for Overflow
           if (sum > MAX_NIT) {
               // Positive Overflow (e.g., 5)
               // Calculate Carry: (5 - 4 + 8)/9 = 1
               int carry = (sum - 4 + 8) / 9;
               
               // Calculate Remainder: 5 - 9 = -4 (Standard wrapping)
               int remainder = sum - (carry * 9);
               
               // Identify Target Dimension
               int next_dim = (i + 1) % NUM_DIMS;
               
               // 3. Look-Ahead Saturation Check (The Circuit Breaker)
               // Can the next dimension accept this carry?
               bool target_is_full = (target[next_dim].is_saturated_positive());

               if (target_is_full) {
                   // DISSIPATION PATH (Thermodynamic Safety)
                   // The carry cannot propagate. Convert to heat.
                   entropy_sink.log_dissipation(carry);
                   
                   // Clamp local value to max (Soft Clipping logic)
                   // We lose the 'wrapping' information to prevent instability
                   target[i].value = MAX_NIT; 
                   
                   // Log the excess local energy as dissipation too
                   entropy_sink.log_dissipation(sum - MAX_NIT);
                   
               } else {
                   // PROPAGATION PATH (Information Conservation)
                   // Target has room. Emit carry.
                   // Local value wraps naturally to keep phase info
                   target[i].value = static_cast<int8_t>(remainder);
                   pending_carries[next_dim] += carry; 
               }
           } 
           else if (sum < MIN_NIT) {
               // Negative Overflow (Symmetric Logic)
               int borrow = (-4 - sum + 8) / 9; 
               int remainder = sum + (borrow * 9);
               
               int next_dim = (i + 1) % NUM_DIMS;
               bool target_is_full = (target[next_dim].is_saturated_negative());

               if (target_is_full) {
                   entropy_sink.log_dissipation(borrow);
                   target[i].value = MIN_NIT;
                   entropy_sink.log_dissipation(MIN_NIT - sum);
               } else {
                   target[i].value = static_cast<int8_t>(remainder);
                   pending_carries[next_dim] -= borrow;
               }
           } else {
               // No Overflow
               target[i].value = static_cast<int8_t>(sum);
           }
       }

       // --- PHASE 2: Apply Carries ---
       // Apply the carries generated in Phase 1. 
       // Mathematical proof guarantees these will NOT generate secondary carries
       // because we only emitted them if the target was unsaturated.
       
       for (int i = 0; i < NUM_DIMS; ++i) {
           if (pending_carries[i]!= 0) {
               int new_val = target[i].value + pending_carries[i];
               
               // Final Sanity Clamp (Defense in Depth)
               // Should not trigger given Phase 1 checks, but prevents memory corruption
               if (new_val > MAX_NIT) {
                   entropy_sink.log_dissipation(new_val - MAX_NIT);
                   target[i].value = MAX_NIT;
               } else if (new_val < MIN_NIT) {
                   entropy_sink.log_dissipation(MIN_NIT - new_val);
                   target[i].value = MIN_NIT;
               } else {
                   target[i].value = static_cast<int8_t>(new_val);
               }
           }
       }
   }
